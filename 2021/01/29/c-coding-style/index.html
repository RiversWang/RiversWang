前言

  马上要入职新公司了,现在学习一下Linux内核的代码风格，为入职以后快速融入打好基础。

1）缩进

  制表符是8个字符，缩进也是8个字符。

  不要在行尾留下空格

  不要用空格来缩进

  不要在一行放多个赋值语句

  对switch语句，让switch和case对齐与同一列

  不要把多个语句放在一行，内核代码风格应该超级简单

  理由：缩进的全部意义在于清楚的定义一个控制卡起止于何处。大一点的缩进更容易分辨。当你需要3级以上的缩进时，你的代码就有问题了，应该立马修改。

2）打散长的行和字符串

  一行代码不要超过80列

  子片段要明显短语母片段，并且明显靠右，对于长参数列表的函数头也是同理。

  不要打散对用户可见的字符串，不然很难grep，比如printk里面的信息

3）大括号和空格的放置

  大括号的放置，首选是把起始大括号放在行尾，把结束大括号放在行首。有一个例外，那就是函数，函数的起始大括号放置在下一行的行头

  结束大括号独自占据一行，除非是do语句的“while”或者是if语句的“else”。

  在只有一个单独语句的时候，不要加不必要的大括号

  当只有一个条件分支是单语句的时候，所有分支都要使用大括号

3.1）空格

  略，以后再码
