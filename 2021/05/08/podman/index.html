1.Container vs VMs

containers比VMs更加轻量化，简单化，和宿主共享内核,低资源占用，可以在不同开发环境之间便携转换。containers适合配置开发环境，便于分享，容易获取。多个开发者不需要多次配置开发环境。

VMs不适合在不同平台间共同使用

而containers+linux内核，方便在所有linux平台上使用

containers并不是运行在dockers上，它是运行在宿主linux 内核上。

2.ADS podman 常用指令:

  1. alias subuildah="sudo /ecs/utils/container/bin/subuildah"

  2. alias supodman="sudo /ecs/utils/container/bin/supodman"（创建alisa）

  3. supodman ps -a (查看所有容器)

  4. subuildah bud -t ubuntu-qsdk(新建image)

  5. sudo /ecs/utils/container/bin/supodman run -u 0:0 –name datetime-test datetime（从image运行容器）

  6. supodman rm datetime-test（删除容器）

  7. supodman rmi localhost/datetime docker.io/library/busybox（删除镜像）

  8. supodman images（查看所有镜像）

  9. supodman run -it -v /nobackup/quahe/sdk/qca-networking-2020-spf-11-3_qca_oem-es1:/home/quahe/qsdk –name magnum ubuntu-qsdk
