使用git工具进行代码管理

一、git介绍（https://git-scm.com/）

  Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

  Git是一款免费和开源的分布式的代码版本控制系统，它被设计用来快速和高效的处理从很小到很复杂的项目。

  Git is easy to learn and has a tiny footprint with lightning fast performance. It outclasses SCM tools like Subversion, CVS, Perforce, and ClearCase with features like cheap local branching, convenient staging areas, and multiple workflows.

  Git易于学习上手，代码少但是有轻快的性能表现。它有类似于Subversion, CVS, Perforce和ClearCase的SCM工具，拥有简单的本地分支，便捷的staging区域（介于add和commit之间的状态）以及多个工作流等特点。

  Git的核心思想是方便你和其他人在同一时间合作工作（版本控制）。现在的项目开发中，大家并行工作，都会在本地原版本的基础上对代码进行修改。

Git相比其他工具的特点

  • 最主要的特征就是：分支与合并
  • small and fast
  • 分布式
  • 数据保证不会出错
  • 免费以及开源

二、git基本操作

###1 远程对github上主版本修改：
$git clone git@github.com:RiversWang/updata-via-network.git

$cd updata-via-network/

$ touch test_git_control.c(新建文件)
$ git add test_git_control.c#add file to staging区域

$ git commit test_git_control.c -m”test to commit message~”(提交，产生版本记录，注意代码依然在本地)

$vim README.md(修改updata-via-network中的README.md文件内容)

$git commit -m”modified README.md”

#提交，产生版本记录，注意代码依然在本地

$git push origin(将修改提交到github上)

###2 远程对github上分支版本修改：

$git clone git@github.com:RiversWang/updata-via-network.git

$git branch test_new_branch (创建新分支)

$git switch test_new_branch (切换到分支中)

$touch new_branch_file.c (在分支中新建文件）

$git add new_branch_file.c (add文件到staging)

$git commit new_branch_file.c (commit提交文件)

$git switch master (从新分支切换到主分支)

$git merge test_new_branch (合并分支到master）
